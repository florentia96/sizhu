06Method

เกณฑ์ตัดสินและขั้นตอน

แยกประเภทความถูกต้อง

  • A · objective — มีคำตอบเดียว (ดาราศาสตร์ ปฏิทิน พิกัด ตำราจีน 八宅 ayanamsa) → เทียบแหล่ง authoritative + ดูว่า test ล็อกค่าที่ แสดง จริงและคลุมครบ
  • B · canonical — คติ ไม่มีคำตอบเดียว (สีมงคล เลขศาสตร์ ฝัน) → เป็น defect เฉพาะเมื่อ (1) ขัดเวอร์ชันนิยม+มี citation, (2) ขัดกันเอง, (3) พังเชิงข้อเท็จจริง · default = ไม่ใช่บั๊ก
  • interpretive — คำบรรยาย/คำแนะนำที่เขียนขึ้น → ตรวจกับแหล่งไม่ได้ ไม่นับเป็น discrepancy เว้นแต่ระบุข้อเท็จจริงที่ผิด

ขั้นตอน

  1. Baseline — รัน typecheck + test ทั้งชุด (689 ผ่าน) เป็นฐานว่าค่าคำนวณรันเขียว → verify: exit 0
  2. Fan-out 8 domain — แบ่งตามแหล่งข้อมูล (ไม่ใช่ตามฟีเจอร์) กัน verify ตารางซ้ำ · แต่ละ finder ดึง claim ที่ แสดงจริง + จัดประเภท + เทียบแหล่ง (WebSearch/WebFetch) + ทำ coverage manifest
  3. Adversarial verify — ทุก discrepancy ถูก skeptic พยายาม ล้ม ("error จริง หรือคนละสำนัก/ปัดเศษ/documented?") → กัน false positive (ผลคือ NUM-01 ถูกปัดตก)
  4. Independent cross-check — ผู้ตรวจหลักยืนยันเอง: 八宅 64 ทิศ, BaZi relations, สูตร kua, กฎ ทักษา 7/8 — เพื่อ break tie
  5. Fix + re-verify — แก้ 3 บั๊กที่ชัด + เพิ่ม regression test → รัน typecheck + test ใหม่ ยืนยันเขียว
✦

หมายเหตุการรัน: finder 2 domain (thai-astro, fengshui-kua) timeout รอบแรก จึงรันซ้ำแยก — ครบทั้ง 8 domain
